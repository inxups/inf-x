# MITE R196 方块破坏等级与专属工具的 26.2 适配

## 审计范围

本实现以仓库参考目录中的 MITE R196 源码为行为基准，重点核对：

- `material/Material.java` 与 `MaterialMinHarvestLevel.java` 的材料最低等级；
- `block/Block.java` 的默认等级继承，以及 `BlockGlass`、`BlockIce`、`BlockLog`、`BlockSandStone`、`BlockSlab`、`BlockStairs`、`BlockOre`、`BlockOreStorage`、`BlockStrongbox`、`BlockRunestone`、`BlockSilverfish` 与各炉体子类覆盖；
- `item/ItemTool.java` 的工具等级公式和有效性判定；
- `ItemPickaxe`、`ItemShovel`、`ItemAxe`、`ItemHoe`、`ItemWarHammer`、`ItemScythe`、`ItemSword` 与 `ItemShears` 的有效材料/方块集合；
- `EntityPlayer.getCurrentPlayerStrVsBlock` 的错误工具不可破坏规则，以及各方块 `isPortable` 的便携豁免。

InfiniteX 不重新分发这些参考源码，也不修改 Minecraft、NeoForge、缓存或依赖 JAR。适配通过 InfiniteX 代码、NeoForge 事件和 Data Generator 标签完成。

## R196 的两个等级

方块等级与工具等级不是同一个字段。

1. 方块构造时默认继承 `blockMaterial.min_harvest_level`。
2. 金属工具的等级等于材料最低等级；非金属工具在非零时减 1；木质材料保持 0。
3. 工具只有同时满足“属于有效工具集合”和“工具等级不低于方块等级”时才有效。
4. 对要求工具的方块，空手、普通物品、错误工具族或等级不足都会得到零破坏强度。

R196 材料结果如下：

| 材料 | 方块材料等级 | 对应工具等级 | 说明 |
|---|---:|---:|---|
| 木 | 0 | 0 | 原木子类额外加到 1，木板仍为 0 |
| 燧石 | 2 | 1 | 岩石材料，工具等级减 1 |
| 石头 | 2 | — | 普通岩石基线 |
| 黑曜石 | 3 | 2 | 岩石材料，工具等级减 1 |
| 锈铁、铜、银、金 | 2 | 2 | 都由 `setMetal(...)` 标记为金属；参数不是“是否金属” |
| 铁、远古金属 | 3 | 3 | 两者采掘能力相同，不是相邻的两个数值等级 |
| 秘银 | 4 | 4 | 金属 |
| 艾德曼 | 5 | 5 | 金属且抗熔岩 |
| 下界岩、石英、红石 | 2 | — | 要求工具的岩石/矿物 |
| 绿宝石 | 3 | — | 晶体材料 |
| 钻石 | 4 | — | 晶体材料 |

因此 InfiniteX 保留材料名工具标签，但实际比较数值为：`flint=1`、`copper=2`、`iron=3`、`ancient_metal=3`、`mithril=4`、`adamantium=5`。铁与远古金属可以互相满足对方的等级要求。

## R196 方块覆盖

R196 在材料默认值上还有以下关键覆盖：

| 方块族 | 等级规则 | 26.2 适配 |
|---|---:|---|
| 原木 | 木材 0 + 1 = 1 | 全部原木、菌柄、竹块及推导的树根为 1 |
| 完整玻璃 | `BlockGlass` 显式 1 | 普通、染色与遮光玻璃为 1；同为玻璃材质的玻璃板、萤石和信标没有该覆盖，仍为 0 |
| 冰 | 材料 0 + 1 = 1 | 冰、霜冰、浮冰与蓝冰统一为 1 |
| 硬化黏土 | 显式 1 | 陶瓦与带釉陶瓦为 1 |
| 砂岩方块 | `BlockSandStone`：石头 2 - 1 = 1 | 普通/红砂岩的完整方块为 1；斧有效且半速 |
| 砂岩台阶 | `BlockSlab` 对砂岩模型显式返回 1 | 普通/红砂岩台阶为 1；源码单独允许斧，且不应用完整砂岩的半速 |
| 砂岩楼梯/墙 | `BlockStairs`/`BlockWall` 重新从石头材质构造，未继承实例覆盖 = 2 | 普通/红砂岩楼梯和墙为 2，仅镐有效 |
| 砂岩炉 | `BlockFurnaceSandstone` 实际使用沙材质 = 0 | InfiniteX 砂岩炉为 0，锹/锄有效，并因炉体便携而可空手回收 |
| 蠹虫方块 | `BlockSilverfish` 实际使用黏土材质 = 0 | 全部原版蠹虫方块和 InfiniteX 蠹虫下界岩为 0；镐、斧、锹有效，锄无效 |
| 刷怪笼 | 显式 2 | 保持 2，且不享受容器便携豁免 |
| 普通矿石 | 构造参数 | 煤、铁、金、青金石、红石、铜、银、石英为 2；绿宝石和秘银为 3；钻石和艾德曼为 4 |
| 储存块 | 材料等级 + 1 | 红石/铜/银/金为 3，铁/远古金属/绿宝石为 4，秘银/钻石为 5，艾德曼为 6；青金石和石英块不是该子类，均为 2 |
| 秘银/艾德曼符文石 | `BlockRunestone` 继承黑曜石本体 = 3 | 两种符文石都为 3；符文金属只影响纹理/传送门，不抬升方块等级 |
| 保险箱 | 材料等级 + 1 | 所有者便携；非所有者执行该等级，详见下文 |
| 不可破坏方块 | 100 | 继续由 26.2 的负硬度/不可破坏行为处理 |

## 有效工具族

“专属工具”表示有效工具集合，不表示每个方块只能有一种工具。R196 明确允许多个工具族重叠。

| 工具族 | R196 有效对象 | 26.2 扩展 |
|---|---|---|
| 镐、战锤 | 艾德曼、远古金属、黏土、煤、铜、珊瑚、钻石、绿宝石、玻璃、金、硬化黏土、冰、铁、秘银、下界岩、黑曜石、石英、红石、石头、锈铁、银和电路 | 以 `mineable/pickaxe` 为现代主体，补玻璃板、萤石、信标、黏土炉/蠹虫块、全部珊瑚，以及火把、梯子、头颅、花盆、按钮、铁轨等 R196 电路材质；活塞是 `piston` 材质，不视为有效 |
| 锹、鹤嘴锄 | 蛋糕、黏土、雪、草、泥土、沙；胡萝卜、洋葱、马铃薯和玻璃板 | 26.2 的锹类松散方块；补黏土炉、蠹虫块和沙材质砂岩炉；甜菜根按根茎作物推导；仅金属锹额外对完整玻璃有效 |
| 斧、短斧、战斧 | 仙人掌、黏土、玻璃、硬化黏土、冰、南瓜、木材；梯子、甘蔗和砂岩 | 保留全部现代斧类木制方块，并补萤石/信标、黏土炉/蠹虫块、玻璃、陶瓦、冰、甜瓜和仙人掌花；只有砂岩完整方块和台阶有效，且只有完整方块半速 |
| 锄 | 蛋糕、雪、草、泥土、沙；胡萝卜、洋葱和马铃薯 | 使用 26.2 松散方块集合但排除黏土；甜菜根按根茎作物推导 |
| 战锤额外集合 | 蛋糕、南瓜 | 26.2 蜡烛蛋糕、南瓜族与甜瓜 |
| 镰刀 | 高草、麦苗 | 保留小麦和草；26.2 新增的非根茎作物、蕨、灌木与干草植被按形态推导，胡萝卜/马铃薯/甜菜根仍归锹/锄 |
| 短木棒、木棒 | 蛋糕、珊瑚、玻璃、冰、南瓜 | 木质工具倍率为 2、等级为 0；因此可处理蛋糕、珊瑚、玻璃板、萤石/信标和南瓜族，但完整玻璃与冰虽属于有效材料，仍因等级 1 而不可采集 |
| 剑、匕首、小刀 | 地毯、布、树叶、植物、南瓜、藤蔓、蛛网 | 补齐现代叶、羊毛、地毯、作物、花、树苗、藤蔓、草、蘑菇、睡莲、可可、下界疣、干草、根系、水生植物、竹和甜瓜族 |
| 剪刀 | 布、树叶、植物、藤蔓、蛛网、绊线 | 同步现代同类植被、羊毛和绊线，但不继承剑独有的地毯/南瓜集合 |

各工具的 R196 基础倍率仍由装备类型决定：镐/斧/锹/剪刀等使用各自原倍率，短斧、战斧、战锤、锄、剑等再应用类型倍率。等级不足或不在有效集合时，物品回落到普通手持速度，但 `BreakSpeed` 会把正等级方块的实际进度归零，最终的破坏事件还会再次拒绝，避免客户端或其他调用绕过等级。

## 挖掘时间与无效点击

R196 的 `EntityPlayer.getDamageVsBlock` 将每 tick 破坏进度计算为：

`strength / hardness / 512`

26.2 对正确工具使用的分母是 30。InfiniteX 因此在 NeoForge `BreakSpeed` 的最终阶段把强度乘以 `30 / 512`，使 26.2 随后的 `/30` 恰好还原 R196 的 `/512`，同时继续兼容物品效率、急迫和其他在事件之前形成的速度。水下与滞空的五分之一惩罚已经由 26.2 计算，事件不再重复乘一次；空腹、麻痹、蛛网和每级 2% 的采掘加成仍按 R196 追加。挖掘疲劳从 26.2 的指数衰减换回 R196 每级减少 20% 的线性强度。

R196 便携方块以 `4 × hardness` 作为最低强度，并在环境惩罚之后再次保底。因此无增益时每个便携方块都是 `4 / 512 = 1 / 128` 的每 tick 进度，即 128 tick（6.4 秒）；等级或急迫可以缩短时间，水下、滞空、空腹等不能把搬取时间拖得更长。保险箱只在所有者或未归属时使用便携速度，外人仍执行材料与工具检查。

对无有效工具、等级不足、最高等级仍无法满足或负硬度不可破坏的目标，实际进度为 0。26.2 原本仍会把这种目标当作正在挖掘，持续播放命中音效并维持方块破坏状态。InfiniteX 在 `Minecraft.startAttack` 与 `continueAttack` 的命中类型读取处使用最小客户端 Mixin，仅把“方块命中且无破坏进度”重定向为 `MISS`。后续仍由原版左键空气分支执行，因此保留一次空挥、攻击冷却重置和 `onEmptyLeftClick`，但不会建立挖掘裂纹、持续命中音效或采掘代谢。服务端同时取消对应的 `START`/`CLIENT_HOLD` 事件，防止异常客户端绕过。

## 26.2 新方块的推导

MITE R196 没有 26.2 的全部注册表。推导按以下优先级执行：

1. 存在直接 R196 方块或材料对应时，采用源码值。
2. 同一材料的新结构继承材料值；九单位密实储存结构在材料值上加 1。
3. 氧化、涂蜡及同一方块的状态/纹理变体不改变等级；台阶、楼梯和墙按 R196 对应构造器处理，不能假定复制原方块实例覆盖。
4. 无材料对应时，先采用相邻 MITE 材料，再以 26.2 的专属工具标签确认工具族。
5. 仍未单列的 `mineable/pickaxe` 方块为等级 2；原木为等级 1；其余方块为等级 0。这使 26.2 注册表中的每个方块都有确定结果。

主要现代推导如下：

| 26.2 内容 | 等级 | 推导依据 |
|---|---:|---|
| 铜矿、铜门/活板门、铜栏杆/链/灯、避雷针 | 2 | 铜材料本体 |
| 铜块、切制/雕纹铜、铜灯泡、铜格栅、铜像及全部氧化/涂蜡变体 | 3 | 密实铜结构，对应铜储存块 |
| 铜箱 | 3 + 便携 | MITE 铜保险箱的底层材料值；26.2 无所有权，因此按普通容器便携 |
| 生铁/粗铜/粗金属块 | 4/3/3 | 对应金属的密实九单位结构 |
| 铁门、铁活板门、铁栏杆、铁链、灯笼、重质测重压力板、重核 | 3 | 铁质结构；重核按密实铁/远古金属推导 |
| 红石块 | 3 | R196 `BlockPoweredOre` 继承 `BlockOreStorage(Material.redstone)`，是明确的“材料 2 + 1”而非普通红石结构 |
| 紫水晶、辰砂、硫磺、凝灰岩、深板岩、黑石、海晶石、树脂砖 | 2 | 石英/石头类矿物和砌体 |
| 珊瑚方块与珊瑚 | 0 | R196 `Material.coral` 为 0，但镐有效 |
| 骨块 | 0 | R196 `Material.bone` 为 0，保留 26.2 镐效率 |
| 压实泥与泥砖 | 1 | 未烧制硬化黏土的保守对应；镐和斧有效 |
| 远古残骸、磁石 | 4 | 下界合金按秘银级材料推导；磁石含下界合金核心 |
| 下界合金块 | 5 | 秘银级材料的密实储存块 |

默认规则不是对未知方块放行：只要第三方或数据包把方块加入原版镐标签，它就自动获得等级 2；若需要其他等级，可显式加入 InfiniteX 等级覆盖标签。

## 便携方块

R196 的 `isPortable` 会绕过底层材料等级。直接对应包括工作台、炉体、箱子、末影箱、发射器、投掷器、酿造台、漏斗、唱片机、音符盒、阳光探测器、附魔台、信标、砧、床、炼药锅、龙蛋、梯子、TNT、告示牌和头颅；刷怪笼明确覆盖为不便携。InfiniteX 同时覆盖 NeoForge `HarvestCheck`，所以“可空手破坏”会真实产出方块，而不是只移除方块却丢失掉落物。

26.2 新容器与工作站按同一语义推导，包括桶、烟熏炉、高炉、潜影盒、Crafter、讲台、织布机、制图台、制箭台、锻造台、切石机、砂轮、钟、蜂箱、饰纹陶罐、雕纹书架、潮涌核心和铜箱。

InfiniteX 保险箱另有所有权规则：

- 所有者或未归属保险箱可空手搬取；
- 非所有者必须使用镐或战锤，并满足“保险箱材料等级 + 1”；
- 与 R196 `BlockStrongbox.dropBlockAsEntityItem` 一致，非所有者即使达到等级并破坏，也不会获得保险箱方块掉落；
- 铜/银/金保险箱要求 3（铁或远古金属以上）；
- 铁/远古金属保险箱要求 4（秘银以上）；
- 秘银保险箱要求 5（艾德曼）；
- 艾德曼保险箱要求 6，现有最高工具等级为 5，因此不能由非所有者破坏。

## 数据与兼容入口

- `infx:requires_harvest_level/0` 至 `/6`：显式等级覆盖；0 用于覆盖原版镐标签的手采方块。
- `infx:effective_tool/pickaxe|shovel|axe|hoe|scythe|cudgel|sword|shears`：有效工具族。
- `infx:effective_tool/metal_shovel`、`war_hammer` 与 `axe_half_speed`：R196 条件性补充。
- `infx:no_effective_tool`：R196 中没有有效工具的结构，目前用于活塞族和砧；砧通过便携规则回收。
- `infx:portable_hand_harvest`：便携豁免。
- `infx:tool_tier/flint|copper|iron|ancient_metal|mithril|adamantium`：第三方工具材料标签，实际比较上述数值而不是枚举顺序。

InfiniteX 自身工具直接执行工具族与等级规则，同时保留现代 `Tool` 组件用于客户端、数据和互操作。第三方工具若要完整参与，应同时提供材料标签，并让 `Tool` 组件对相应 `effective_tool` 标签返回正确工具能力。

## 验证边界

单元测试锁定等级公式、铁/远古金属同级、显式覆盖优先级、`30 / 512` 换算、128 tick 便携基线、挖掘疲劳和无效目标到 `MISS` 的映射；生成资源测试锁定 0–6 标签、26.2 铜变体、砂岩构造差异及有效工具标签；GameTest 覆盖红石储存块、符文石、蠹虫黏土材质、砂岩方块/台阶/楼梯差异、全部工具族代表、普通/便携挖掘进度、无效点击与代谢、空手煤块掉落、等级 3–6 边界和保险箱所有权/掉落，并遍历完整方块注册表，保证每个正等级可破坏方块都有有效工具族或便携入口，且不存在多个显式等级。
