# MITE R196 合成难度与合成等级的 26.2 适配

## 源码依据

本实现以仓库参考目录中的 MITE R196 源码为准，核心依据如下：

- `net/minecraft/recipe/RecipeHelper.java`：配方难度等于各占用槽位组件难度之和，并决定是否检查工具台材质硬度。
- `net/minecraft/entity/EntityPlayerSP.java`：基础合成周期、玩家等级、工具台加速和 Clumsiness 的运算顺序。
- `net/minecraft/entity/EntityPlayer.java`：品质经验成本及 Clumsiness 的成本惩罚。
- `net/minecraft/ReferenceFileWriter.java`：组件难度和配方难度的定义说明。

Minecraft 26.2 使用标签 Ingredient、动态输出特殊配方及数据包配方。InfiniteX 因此保留 R196 的数学规则，但在服务器匹配到具体输入后动态计算普通 26.2 配方；JEI 无具体输入时使用标签中最低组件值生成代表值。

## 配方难度

每个占用槽位计算一次组件难度，不乘槽内堆叠数量：

```text
recipeDifficulty = max(25, sum(componentDifficulty(each occupied slot)))
```

组件值按以下优先级确定：

1. R196 源码明确值，例如木棍/纸/煤炭为 25，皮革为 100，烈焰棒为 200，燧石为 100，黑曜石块为 240。
2. R196 材料耐久公式：金属锭为 `materialDurability × 100`；铜/银/金 400、铁 800、远古金属 1600、秘银 6400、艾德曼 25600。
3. 粒为对应锭的 `1/9`；链为对应锭的 `4/9`；碎片通常为完整材料的 `1/9`，燧石碎片为 `1/4`。
4. 金属/宝石储存块按组成材料计算；例如铁块 7200、钻石块 14400、石英块 3600。
5. 其他方块使用 26.2 `defaultDestroyTime × 100`，最低 25；不可破坏方块采用 25600 的封顶推导值。
6. R196 没有直接对应关系的普通 26.2 物品使用保守叶值 25，保证没有免费或即时合成路径。

现代材料向最近的 R196 工作台能力映射：

| 26.2/扩展材料 | 工作台等级 | 依据 |
| --- | --- | --- |
| 木材、皮革、燧石 | 燧石 | 能力 1，木材按 R196 木工门槛 |
| 黑曜石 | 黑曜石 | 能力 2 |
| 铜、银、金、青铜、石英 | 铜 | 能力 4 |
| 铁、钢、绿宝石 | 铁 | 能力 8 |
| 远古金属、钻石 | 远古金属 | 能力 16 |
| 秘银 | 秘银 | 能力 64 |
| 艾德曼、下界合金 | 艾德曼 | 能力 256 |

钻石、绿宝石和石英不是凭名称随意升级，而是分别按 R196 材料耐久 `16/8/4` 映射。下界合金没有 R196 对应物，按高于秘银的现代终局材料映射到艾德曼；其现有禁用配方仍保持禁用。

## 合成周期与玩家等级

R196 基础周期：

```text
base =
  difficulty < 25   -> 25
  difficulty <= 100 -> round(difficulty)
  otherwise         -> round((difficulty - 100)^0.8) + 100
```

最终周期：

```text
ticks = max(
  25,
  floor(base × (Clumsiness ? 2 : 1)
        / (1 + playerLevel × 0.02 + benchBonus))
)
```

每个显示等级提供 2% 合成速度。Clumsiness 必须在除以等级和工具台加成前翻倍，最后再应用 25 tick 下限，因此不会错误地形成 50 tick 下限。

## 工具台等级与加速

| 工具台 | 能力 | 材料受限配方加速 |
| --- | ---: | ---: |
| 手工 2×2 | 0 | 0% |
| 燧石 | 1 | 20% |
| 黑曜石 | 2 | 20% |
| 铜/银/金 | 4 | 30% |
| 铁 | 8 | 40% |
| 远古金属 | 16 | 50% |
| 秘银 | 64 | 60% |
| 艾德曼 | 256 | 70% |

普通配方只按 2×2/3×3 网格决定最低环境；只要在任意工具台上合成，始终使用固定 20% 工具台加速。材料受限配方才使用实际工具台的 20–70% 加速。

材料门槛以配方输出为主，避免仅因红石或轨道配方含有铁锭就错误升级。金属装备、护甲、储存块、门、活板门、栏杆、链、砧等现代材料制品按输出材料检查。R196 的锭/粒转换、碎片/宝石转换、金属小刀以及以另一个桶为组件的桶转换不附加材料硬度门槛。木板是 R196 的明确特例，需要燧石级木工工具。

## 工具品质选择

所有带耐久的 InfiniteX 装备都有 Average（无组件）及 Wretched、Poor、Fine、Excellent、Superb、Masterwork、Legendary 品质。非平均品质会持久化并同步，调整成品耐久；可选择的最高品质仍受装备材料限制。

在原版 3×3 工作台或 InfiniteX 工具台中，将工具配方放入网格后，**右键结果槽**即可循环品质。循环优先尝试 Fine 及更高品质，跳过材料不支持或当前总经验不足的品质，随后可选择免费 Wretched/Poor，最后回到 Average。右键后结果预览和聊天提示会同步显示所选品质及完成时将扣除的经验；左键结果槽才开始限时合成。

Fine 及更高品质的经验成本为品质调整后配方难度的 `round(difficulty / 5)`；Clumsiness 会将该成本翻倍。服务端会在选择预览和成品完成两个时点校验总经验，并仅在成功产出时扣除准确成本。

## 26.2 全量接入

- InfiniteX 显式限时配方继续使用数据生成器中已经按 R196 恢复的难度和工具台等级。
- `minecraft` 命名空间下所有类型路径以 `crafting_` 开头的配方 JSON，会通过 NeoForge 公共 `ModifyRecipeJsonsEvent` 在反序列化前删除；这覆盖 26.2 的有序、无序、特殊、转化、染色、注魔和饰纹陶罐配方，不依赖静态配方 ID 清单。
- 原版熔炼、烧炼、烟熏、营火烹饪、切石和锻造配方不在批量删除范围内；已有 progression 禁用项继续由 `DisabledVanillaRecipesProvider` 管理。
- 服务器仍加载的第三方或数据包 `RecipeType.CRAFTING` 配方自动进入同一限时引擎，标签和特殊配方使用实际槽位动态计算。
- 2×2 物品栏、九种 InfiniteX 工具台及原版 3×3 工作台均只显示仍存在的限时配方；原版 3×3 工作台可继续承载 InfiniteX 替代配方。
- 原版 3×3 工作台没有可替换结果计算的公共事件，因此使用仅针对 `CraftingMenu` 结果刷新和数据槽的最小 Mixin；不修改 Minecraft 或 NeoForge 文件。
- 剩余普通配方和 InfiniteX 配方都会同步到客户端。JEI 按推导后的最低工具台分类，显示代表难度与所需工作台，并可转移到能力足够的分级工具台。

品质合成仍使用品质调整后的难度。Clumsiness 除翻倍合成周期外，也会按 R196 翻倍品质经验成本；选择和服务端最终校验使用相同成本，避免客户端可选但完成时失败。

## 验收

GameTest 会扫描服务器实际加载的每一条剩余普通 crafting 配方，要求其代表难度为正且有限、工作台等级可由艾德曼工具台支持；显式 InfiniteX 配方执行同样检查。行为测试同时覆盖：

- 服务器最终不得加载任何 `minecraft` crafting 配方，原版火把配方必须不存在；
- 原版非 crafting 配方仍须存在，避免批量规则越界；
- 原版 3×3 `CraftingMenu` 必须能够限时执行 InfiniteX 替代配方；
- 位于 3×3 网格偏移位置的替代配方完成后必须消耗正确槽位；
- 现有禁用配方仍未加载。
