# InfiniteX R196 Roadmap

更新基准：2026-07-18，`codex/r196-complete`。

## 已完成：高级生存链与 Underworld

- [x] 注册 `infx:underworld` 的 Level、LevelStem、DimensionType、Biome 与 NoiseGeneratorSettings。
- [x] 使用 `Y=-192–319` 高度、`Y<0` 深板岩与 `Y=0–127` 石头分层、`Y=140` 水位、无天空光且有顶棚的固定地下环境。
- [x] 加入基础地下怪物、洞穴蜘蛛、两类怪物房、蘑菇、普通地下矿物、银矿、秘银矿与 Underworld 专属艾德曼矿。
- [x] 艾德曼矿脉大小 3，底部偏置分布，平均每区块 8 次候选；不向主世界或下界添加生成。
- [x] Underworld 普通地牢加入远古金属粒/锭、硬币、工具、弓、锁链甲与远古金属马铠。
- [x] 主世界底部接触基岩的黑曜石门进入 Underworld，返回位置持久化；两端扫描安全点，失败时构建平台和门框。

依赖：秘银镐、热量 4 下界岩炉、艾德曼矿石与远古金属装备目录。验收：数据注册表可加载，Underworld 作为服务端维度存在，门可双向解析目标，艾德曼只存在于 Underworld 生物群系生成设置，地牢能产生远古金属阶段物品。

## 已完成：装备材料特殊行为

- [x] 银制近战和银箭只对亡灵造成 1.25 倍伤害。
- [x] 木弓保持标准弹速；远古金属弓和秘银弓为 1.10 与 1.25 倍。
- [x] 艾德曼矿石、原材料、粒、锭、锁链、硬币、工具、武器、护甲和砧物品均抗火焰/熔岩。
- [x] 十种材料箭按 30/40/60/60/50/50/70/80/80/90% 执行一次性回收，创造/无限路径不复制箭。
- [x] 玩家板甲和锁链甲在剩余耐久低于 50% 后按 `remaining × 2` 线性衰减；马铠保持固定保护。

验收：普通目标无银加成；箭保留材料、基础伤害与拾取物；护甲在完整、50%、临界与修复后得到预期属性；非艾德曼物品不获得抗火组件。

## 已完成：品质、修复与金属砧

- [x] 品质数据组件支持持久化与网络同步；Average 用无组件表示。
- [x] 实现 Poor、Fine、Excellent、Superb、Masterwork、Legendary 与材料最高品质限制。
- [x] 品质影响耐久和合成时间，提示使用品质颜色；修复保留品质、名称、附魔和其他组件。
- [x] 注册铜、银、金、铁、远古金属、秘银、艾德曼砧及四种配方依赖的金属储存块。
- [x] 金属砧拥有朝向、三阶段外观、持久损伤、下落数据保留、带损伤掉落、专用菜单和服务端重算。
- [x] 工具/武器和板甲完整修复成本为组件数 ×2，锁链甲为组件数，强化弓为 2，金属鱼竿为 1；高等级装备拒绝低等级砧。
- [x] 32 张新增批准方块材质（28 张砧、4 张储存块）已纳入来源清单。

边界：现代版本不恢复已移除的 R196 技能系统；专用砧修复不收取经验，品质不降级，附魔不由修复流程新增或合并。

## 已完成：锈铁、硬币与目录原材料

- [x] 锈铁装备默认 Poor 品质和 0.75 倍耐久。
- [x] 锈铁链与工具进入普通地牢/废弃矿井，僵尸可持有锈铁工具/板甲，骷髅提供锈铁箭。
- [x] 铜、银、金、远古金属、秘银和艾德曼硬币分别兑换 5、25、100、500、2500、10000 经验；只在服务端消费一次。
- [x] 肥料由牛/哞菇、猪/羊、鸡按 24000、48000、384000 tick 周期产生，并复用现代可施肥方块契约。
- [x] 钻石碎片来自主世界沙砾稀有分支；同一分支在下界沙砾产下界石英碎片；玻璃块/板提供玻璃碎片。
- [x] 钻石、下界石英、玻璃板与对应碎片支持 1↔9 转换。

排除项：R196 没有铁物品按湿度、环境、物品栏时间自动转成锈铁的实现，因此不保存或推进虚构“腐蚀进度”。若未来出现新的可核验 R196 证据，再单独设计迁移与持久化规则。

## 明确排除

- 马铠制造配方；秘银/艾德曼马铠的虚构战利品来源。
- 钻石工具、武器和护甲；金属小刀；石匕首；`chip_flint_knife`；铁币。
- Wretched 品质（当前需求只保留六个非平均持久品质，Average 保持兼容表示）。
- 未经批准来源的材质，以及对 Minecraft、NeoForge、Gradle 缓存或依赖 JAR 的直接修改。

## 持续验收

每次相关变更必须同步本文件和 `CHANGELOG.md`，并执行 `bash gradlew test`、`bash gradlew runData resourceTest`、`bash gradlew runGameTestServer`、`bash gradlew build`。新增数据生成资源必须修改 Provider，不只修改生成结果。
